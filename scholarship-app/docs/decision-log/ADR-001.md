# ADR-001

Area: Architecture
Context: External databases introduce high network latency and complex configuration. Edge functions on serverless models frequently hit read-write concurrent locks.
Date: June 28, 2026
Decision Name : SQLite-in-Repo Pattern for Edge Rendering
Status: Accepted

### Context

External databases introduce high network latency and complex configuration. Edge functions on serverless models frequently hit read-write concurrent locks.

### Conversational Rationale

We decided to keep it simple and blazing fast. We store our entire SQLite database (data/scholarships.db) right inside the repository. This means when Vercel builds the site, it queries the database locally. For edge functions, the file is read-only, which completely avoids concurrent write locks. The trade-off is that every database update requires a git commit and push, but given our low update frequency, this is a massive win for performance and hosting costs.
