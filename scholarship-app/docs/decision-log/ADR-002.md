# ADR-002

Area: Architecture
Context: Content writers require a visual CMS editor (WordPress), but developers need a fast React codebase without blocking database hooks or latency issues.
Date: June 28, 2026
Decision Name : Headless WordPress CMS with SQLite Fallback
Status: Accepted

### Context

Content writers require a visual CMS editor (WordPress), but developers need a fast React codebase without blocking database hooks or latency issues.

### Conversational Rationale

We chose a hybrid headless CMS approach. Writers add and edit listings in a standard Hostinger-hosted WordPress setup. Next.js pulls this content via the REST API, caching it hourly. To make sure the site NEVER goes down, we built a fallback: if the WordPress server fails or has latency spikes, Next.js instantly falls back to reading the local SQLite database in the repo. This gives us the editing convenience of WordPress with the bulletproof reliability of static files.
