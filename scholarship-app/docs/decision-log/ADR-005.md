# ADR-005

Area: Architecture, Product
Context: AtlasOS charter/handoff (Next.js + Drizzle + Tiptap + dnd-kit + custom MCP server) specced a full local-first PM app to replace Notion. Reviewed via Claude critique: risk of scope creep for a non-coder solo founder already at WIP=5 on IndiaScholarships, plus data model issues (parent_id and links table both encoding hierarchy; generic Item schema doesn't fit Decision/Bug/Feature well) and missing pieces (no Notion migration plan, no backup strategy, no FTS mechanism named, no concurrency handling for multiple AI agents writing to one SQLite file). Revised direction: V1 = simple database(s) with a UI where both Roshan and AI agents can write/modify tasks, like a lightweight task planner. Documentation stays in local Markdown files.
Date: July 9, 2026
Decision Name : AtlasOS V1 scope: simplify from full custom app to local task DB + MD docs
Status: Proposed

## Decision

AtlasOS V1 is scoped down from the original charter to: a simple task-planner database with a UI, readable/writable by both Roshan and AI agents via MCP. Full-featured PM system (Kanban, relationships, rich-text notes, tags, multi-type Items) is deferred.

Documentation continues to live in local Markdown files, not inside AtlasOS.

## Open items for next pass

- Resolve hierarchy: pick either parent_id or the links table, not both
- Decide backup/export strategy for the SQLite file
- Name the actual search mechanism (e.g. SQLite FTS5) behind search
- Decide concurrency handling if multiple AI agents write to the DB at once
- If AtlasOS is meant to eventually replace the IndiaScholarships Notion brain, a migration plan is still needed

## Status

Proposed - not yet built. Full original charter and engineering handoff kept as reference for post-V1 scope.
