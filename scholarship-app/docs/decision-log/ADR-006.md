# ADR-006

Area: Architecture, Project Management
Context: To avoid API dependency, credit consumption, and connectivity issues (such as Notion integration permission failures), we need to transition away from cloud Notion database tracking (`IS Dev Backlog` and `IS Content Backlog`) for IndiaScholarships.
Date: July 9, 2026
Decision Name: Transition to local JSON/Markdown backlog and Kanban board frontend, deprecating Notion task sync
Status: Accepted & Implemented

## Decision

We have fully migrated the IndiaScholarships project backlogs offline and into the repository:
1. **Local Backlog Storage**: Exclusively store task lists inside local JSON databases (`data/backlog-dev.json` and `data/backlog-content.json`) to act as the single source of truth.
2. **Markdown Mirror**: Automatically generate and synchronize human-readable Markdown files (`data/backlog-dev.md` and `data/backlog-content.md`) whenever the JSON backlog updates. This ensures clear diffs in Git.
3. **Local Kanban Dashboard**: Built an interactive Kanban UI inside the Next.js admin area at `/admin/backlog` to allow visual tracking, columns sorting (Backlog, In Progress, Done, Parked), task CRUD, and search.
4. **AI-Agent Integration via MCP**: Configured the official `@modelcontextprotocol/server-filesystem` MCP server in the Claude Desktop app settings pointing to the project root. This enables Claude Desktop to read/write these backlog files directly using standard filesystem tools.
5. **Dashboard script deprecation**: Refactored the local dashboard generator (`scripts/update-notion-dashboard.js`) to generate a local `data/dashboard.md` using the local JSON backlog. Direct Notion sync is bypassed but kept as a silent warning-safe fallback.

## Consequence

- **Pros**: Zero Notion API credits consumed. Fast, local, offline access. Full git history tracking of all task changes. Immediate updates on local Next.js server.
- **Cons**: AI assistants running on standard web interfaces (like browser-based ChatGPT/Claude) cannot write directly to the local filesystem without manual file upload/download or custom API tunnels (like ngrok).
